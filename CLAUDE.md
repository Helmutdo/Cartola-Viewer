## What This Is

Browser-only (no backend) PDF reader for **Banco Falabella** (Chile) account statements ("cartolas"). Extracts transactions from PDFs, categorizes them, and visualizes spending. All data lives in `localStorage`.

## Architecture

### Data Flow

```
PDF file
  → src/lib/pdfText.ts          (pdfjs-dist → raw text string)
  → src/lib/parseFalabella.ts   (text → ParsedRow[], cat assigned via categorizer.ts)
  → src/lib/merchants.ts        (resolveMerchant → display name enrichment, display-only)
  → src/store/useCartola.ts     (addPdfFiles → MonthData[] → localStorage)
```

### Merchant Normalization Layer

`resolveMerchant(desc, userMerchants)` in `src/lib/merchants.ts`:
1. Search user-defined `MerchantAlias[]` from store (priority)
2. Fall back to `COMMUNITY_MERCHANTS` bundled constant (`src/data/communityMerchants.ts`)
3. Returns `MerchantAlias | null` — **display-only**, does not modify `Transaction.cat`

`MerchantAlias` maps legal company names → friendly display names:
- `"WALMART CHILE ..."` → `"Lider"`
- `"SOCIEDAD DE HERMANOS... MULTICAR SPA"` → `"Unicar"` (user-defined)

Fields: `id`, `displayName`, `patterns: string[]`, `defaultCategory?`, `source: 'user' | 'community'`, `confidence?` (reserved M4+), `createdAt`, `updatedAt`.

Stored in `localStorage` key `cartola:merchants`. Community merchants bundled in `src/data/communityMerchants.ts` (~60 Chilean merchants), never stored.

### Categorization Pipeline

`categorize(desc, rules)` in `src/lib/rules.ts`:
1. `applyRules(desc, rules)` — user rules in order, first match wins (contains / startsWith / exact, case-insensitive)
2. Fall back to `autoCategorizeFallback(desc)` — hardcoded keyword patterns

`src/lib/categorizer.ts` is a shim: re-exports `autoCategorizeFallback as autoCategorize` from `rules.ts`. `parseFalabella.ts` imports from it.

Two override layers stored separately:
- **prefix override** (`cartola:prefixOverrides`) — `desc.slice(0,20)` as key; `applyCategoryToDesc` writes here and patches all matching transactions
- **transaction override** — `catOverride` field on a single `Transaction` (written by `applyCategoryToTransaction`)

`effectiveCategory(t)` = `t.catOverride ?? t.cat` — always use this for display/aggregation.

### State

Single Zustand store: `src/store/useCartola.ts` (`useCartola`).  
Call `hydrate()` once on mount to load from localStorage. All writes go through store actions which persist automatically.

Key actions:
- `addPdfFiles(files)` — parse PDFs, merge by month key, persist
- `applyCategoryToDesc(desc, cat)` — bulk prefix override
- `applyCategoryToTransaction(id, cat)` — single transaction override
- `addRule / removeRule / updateRule / reapplyRules` — manage `CategoryRule[]`
- `setCategoryTree(tree)` — replace full tree
- `renameMainCategory(old, new)` — migrates budget keys
- `renameSubcategory(old, new)` — patches `cat`/`catOverride` across all transactions
- `removeSubcategories(names)` — resets affected transactions to `'Sin categorizar'`
- `addMerchant / updateMerchant / removeMerchant` — CRUD on user merchants
- `setBudget(cat, limit)` — set or clear a budget limit
- `resetAll()` — clears localStorage, regenerates default category tree

### Storage Keys (localStorage)

| Key | Content |
|-----|---------|
| `cartola:months` | `MonthData[]` (all transactions) |
| `cartola:budgets` | `BudgetMap` (cat → limit) |
| `cartola:merchants` | `MerchantAlias[]` (user-defined only) |
| `cartola:prefixOverrides` | `Record<prefix, SubCategory>` |
| `cartola:dismissed` | dismissed alert keys |
| `cartola_rules` | `CategoryRule[]` |
| `cartola_category_tree` | `CategoryTree` |
| `override:<desc>` | legacy per-desc overrides |

### Types

- `CategoryTree = CategoryDefinition[]` — each `CategoryDefinition` has `{ id, name, color, subcategories: SubCategoryDefinition[] }` where `SubCategoryDefinition = { id, name }`. Default tree generated by `makeDefaultCategoryTree()` in `storage.ts` on first load.
- `MerchantAlias` — see Merchant Normalization Layer above
- `Transaction.cat` — auto-assigned at parse time; `Transaction.catOverride` — manual override
- `SubCategory` / `MainCategory` are plain `string` (dynamic, not enums)
- `getMainCategory(sub, tree)` — resolves a subcategory name to its parent `CategoryDefinition.name`
- `effectiveCategory(t)` exported from `src/types/index.ts`
- `CategoryRule` — `{ id, pattern, matchType: 'contains' | 'startsWith' | 'exact', category, createdAt }`
- `ParsedRow` — `{ fecha, desc, cargo, abono, month, cat }` (cat assigned during parsing)

### PDF Parsing Notes

`parseFalabella.ts` handles two Falabella line formats:
- 2 amounts per line → `[amount, saldo]`; cargo vs abono determined by `isAbonoByDesc()`
- 3+ amounts per line → `[cargo, abono, saldo]`

Chilean amounts use `.` as thousands separator (e.g. `1.234.567`), parsed via `parseChileanAmount()`.  
Description noise stripped by `NOISE_RE` (`Sucursal Virtual NNN`, `CASA MATRIZ`).
